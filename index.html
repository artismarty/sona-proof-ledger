How to verify this yourself

  1. Clone the public proof-ledger repository (this page ships inside it) — its git commit history is the independent timestamp: each commit proves the hashes it contains existed no later than the commit time.
  2. Check the chain in ledger.jsonl: every entry's entry_hash is the SHA-256 of its canonical JSON (sorted keys, compact separators, entry_hash field removed), and each entry's prev_hash must equal the previous entry's entry_hash. Edit or remove any line and every later hash breaks.
  3. When a prediction payload is revealed, re-hash the revealed canonical JSON file (SHA-256) and compare it to the artifact sha256 committed for that day — a match proves the prediction existed, unaltered, on the committed date.
